Malvoz
2018-08-26

Initially brought up in https://github.com/w3c/html/issues/1333 - Should logical sizing: inline-size | block-size exist as HTML attributes in addition to the width and height attributes for the purpose of internationalization?

This was briefly discussed and @prlbr mentioned that CSS logical properties:

[…] correspond to the width and height properties. The mapping depends on the element’s writing-mode.

If logical sizing attributes are desired, I think they’d depend on a - yet to be defined - writing-mode attribute.
