Network Error Logging intro articles and request for feedback

dcreager
2018-04-03

We’ve been making a lot of progress recently on the Reporting and Network Error Logging specs over the last few months and we’re ready to start spreading the word about them more widely.

To that end, I’m going to start writing some intro blog posts about these two specs, so that we have some content for people who don’t want to read dry specification text.

The first one is live — a basic intro to Network Error Logging. I’d appreciate any feedback about that post.

And more importantly, please let me know if there is anything else in particular that you’d like me to focus future articles on! I’d also love to hear if there are any operational issues or outages that this kind of client-side monitoring/observability would have let you diagnose and debug more easily.

Thanks!
