Streams for the proposed WebRTC QUIC API

lgrahl
2018-09-12

Some WebRTC spec folks are currently working on an extension to be able to use QUIC streams in WebRTC.

But here’s the thing: They currently roll their own streaming API for it. Perhaps I’m misguided or I’m just alone with that perception but isn’t this a perfect fit for the streams standard? So, last year I created an issue and wrote a prototype for it to use that API. Since then, not much has happened. So, if you agree (or don’t agree) that this technology should leverage the streams standard, please add your two cents in that issue and maybe they will eventually reconsider.

Cheers!
