[Proposal] Add new Named Character References for dates (e.g. &year;)

camdyn
2023-04-17

Perhaps this goes against the philosophy of character references being static values, but it would be nice to be able to insert a date with a simple character reference - such as when inserting the copy write year in the footer of a website.

It just seems that having to insert this value with a build process or with JavaScript is overkill.

<footer>&copy; Company Inc. &year;<footer>

Would love to get some input on this, just a thought I had.
